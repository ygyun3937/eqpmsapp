## 0장. 관리자 사전 준비 (강의 전, ADMIN 1회)

### 슬라이드 0-1. 사전 준비 체크리스트
- 시스템 설정 → **Google Drive 루트 폴더 연결 + 연결 테스트 OK**
- 시스템 설정 → **주간 업무 보고 기능 ON** → 전체 설정 저장
- 사용자 관리 → 직원 ID/비밀번호 발급 + **부서(dept)** 입력
- 사용자 관리 → 부서장에게 **"팀장" 토글 ON**
- 사용자 관리 → 사용자별 **"주간 보고" 토글 ON** (필요 인원만)
- **노트:** 강의 시점에 위 항목이 모두 끝나 있어야 실습 진행 가능. dept 값이 비어 있으면 팀장-부서원 매칭이 안 됨.

## 1장. 표지 / 도입

### 슬라이드 1-1. 표지
- **EQ-PMS — 장비 프로젝트 셋업 관리 시스템**
- 사용자 교육 / 신규 기능 안내
- 발표자 / 일시 / 부서
- **노트:** 회사 로고 + 시스템 스크린샷(대시보드) 1장 배경.

### 슬라이드 1-2. 오늘 배우는 것
- 시스템이 무엇을 해결하는지 (3분)
- 매일 쓰는 7가지 화면 (15분)
- 한 주에 한 번 쓰는 주간 보고 (10분) ★ 신규
- 권한별로 무엇을 할 수 있나 (5분)
- 보고서 추출 / 자주 묻는 질문 (10분)
- **노트:** 이번 라운드 핵심은 "주간 업무 보고 자동 작성"과 "간트/일정 모달 통일". 이 두 가지에 시간 비중 두기.

### 슬라이드 1-3. 한 줄 요약
- 프로젝트 셋업의 **모든 기록을 한 곳에**, 보고는 **자동으로** 받기
- (좌) 기존 — 엑셀/워드/메일/카톡 분산 → (우) EQ-PMS — 단일 시스템
- **노트:** 기존 운영을 비판하지 말고, 시간이 절약된다는 점을 강조. "주간 보고 작성 90% 단축" 데이터 활용.

## 2장. 시스템 개요

### 슬라이드 2-1. EQ-PMS는 무엇인가
- 반도체 / 디스플레이 / 2차전지 장비 셋업 프로젝트 전용 PMS
- 본사 관리자 · 현장 PM · 엔지니어 · 고객사가 같은 화면을 공유
- Google Drive + Google Sheets 기반 — 별도 서버 불필요
- 모바일/PC 자동 전환 (768px 미만 = 모바일 모드)
- **노트:** "다른 PMS와 다른 점은 산업·도메인 특성에 맞춘 검수표 / 간트 / 자격(출입증·안전·비자) 관리"

### 슬라이드 2-2. 메뉴 구조 (좌측 사이드바)
- 대시보드 / 프로젝트 관리 / 이슈·펀치 / AS 통합 관리 / 자재 관리
- 사이트·유틸 마스터 / 인력·리소스 / 버전 릴리즈 / 사용자 관리(ADMIN)
- 시스템 설정(ADMIN) / **주간 업무 보고** (활성화·권한 보유 시)
- 사이드바 하단 **"메뉴 접기"** 버튼 → 컨텐츠 영역 확대
- **노트:** 권한·기능 활성화 여부에 따라 메뉴 노출 자동 조정.

### 슬라이드 2-3. 데이터는 어디에 저장되나
- 모든 변경 → **Google Sheets DB**에 자동 저장
- 첨부 파일 → **Google Drive**의 `[루트]/[고객사]/[프로젝트]/[카테고리]` 폴더로 자동 정리
- 카테고리: 명세서 / 도면 / 회의록 / 기타
- **노트:** "엑셀 파일 따로 관리 안 해도 된다 = 버전 충돌 없다"가 핵심 메시지.

## 3장. 시작하기

### 슬라이드 3-1. 로그인 / 비밀번호
- 관리자 발급 ID/비밀번호로 로그인
- 최초 접속 시 비밀번호 변경 강제
- 우측 상단 키 아이콘 → 언제든 비밀번호 변경
- **노트:** 비밀번호 분실은 ADMIN에게 요청 (재설정 후 다시 1회 강제 변경).

### 슬라이드 3-2. 헤더 아이콘 안내
- 🔔 종 아이콘 — **알림 센터** (전사 신규 활동 모아보기)
- 🌐 모드 전환 — PC ↔ 모바일
- 🔑 비밀번호 변경
- ❓ 도움말 (이 가이드)
- 🚪 로그아웃
- **노트:** 알림 센터에서 회사 전체 신규 활동을 한눈에. 직접 시연.

### 슬라이드 3-3. 알림 센터 (🔔)
- 포함: 공유 노트 / 신규 이슈 / 고객 요청 / AS / 버전 / 출장 / 추가 작업
- **미확인 건수 빨간 뱃지 + 항목 클릭 시 해당 화면으로 점프**
- 필터 칩: 전체 / 미확인 / 종류별
- "모두 읽음" 버튼으로 일괄 처리, 마지막 본 시각은 사용자별 저장
- **노트:** 출근 직후 5초 동안 종 아이콘만 봐도 어제 전사에서 무슨 일이 있었는지 파악 가능.

## 4장. 권한 체계 (4 Role + 1 부가 권한)

### 슬라이드 4-1. 4개 기본 역할
| 역할 | 누가 | 핵심 |
|---|---|---|
| **ADMIN** | 본사 관리자 | 모든 기능 + 사용자 관리 + 사인 취소 |
| **PM** | 현장 PM | 프로젝트/이슈/자재/AS 전반, 담당자 변경 |
| **ENGINEER** | 현장 엔지니어 | 본인 배정 프로젝트의 진행 입력 |
| **CUSTOMER** | 고객사 | 자기 회사 프로젝트 조회 + 요청 + 검수 서명 |
- **노트:** 역할은 사용자 관리에서 ADMIN이 지정. 한 명당 하나만 부여.

### 슬라이드 4-2. 부가 권한 — 팀장 (NEW)
- 기본 역할(PM/ENGINEER 등)과 **별도로 부여**
- 사용자 관리에서 **"팀장" 컬럼 토글**
- 권한: 같은 부서(dept) 부서원의 **주간 보고 검토/승인/반려 + 팀 종합 보고서 작성**
- ADMIN은 전사 자동 권한
- **노트:** "PM = 프로젝트 책임자, 팀장 = 부서장. 같은 사람이 둘 다일 수도, 분리될 수도 있다."

### 슬라이드 4-3. 권한 한눈에 보기 (요약표)
- 가이드 모달 → "권한별 기능" 탭에 32행 전체표 제공
- 본 슬라이드에서는 자주 쓰는 7가지만:
  - 프로젝트 생성 — ADMIN/PM
  - 검수 사인 — CUSTOMER (ADMIN 대리)
  - 사인 취소 / Buy-off 후 수정 — ADMIN 단독
  - 도메인(산업군) 수정 — ADMIN 단독
  - 주간 보고 검토 — 팀장(같은 부서) / ADMIN(전사)
  - 자재/스페어/AS 등록 — ADMIN/PM/ENGINEER 공통
  - 시스템 설정 / 사용자 관리 — ADMIN 단독

## 5장. 대시보드

### 슬라이드 5-1. 대시보드 한눈에
- 상단 **KPI 카드 5개 (모두 클릭 가능)**
- 좌측 — 전체 프로젝트 일정(간트) + 인력 배치 사이드바
- 우측 — **이슈 / AS 통합 위젯 (상하 2단)**
- 하단 — 도메인·진척률 분포 등 보조 지표
- **노트:** 카드 5개·위젯 1개·간트 1개 — 총 7개 영역만 기억하면 됨.

### 슬라이드 5-2. KPI 카드 5장
1. **미해결 이슈** (orange) — 심각도순 팝업
2. **지연·위험** (rose) — 위험점수순 프로젝트
3. **임박 마일스톤 30일** (amber) — D-day 순
4. **고객 요청 미처리** (cyan) — 긴급도순, 클릭 시 해당 프로젝트 고객요청 탭
5. **평균 진척률 (진행중)** (emerald) — delta 뱃지 + 4버킷 분포 + 계획 위치 마커
- **AS는 KPI 카드 X — 우측 위젯에서 추적**
- **노트:** "신규 착수"는 KPI에서 빠지고 분석 모달에서 시계열로 본다.

### 슬라이드 5-3. 우측 위젯 — 이슈 / AS
- **상단 — 미해결 이슈**: severity 가로 스택 + 최근 3건. 헤더 클릭 = 전체 팝업
- **하단 — AS 미완료**: 유형(긴급출동/정기점검/부품/불량/보증) 막대 + 최근 3건. 헤더 클릭 = 긴급출동 우선 팝업
- **노트:** AS는 회사 입장 critical 지표지만 KPI 자리는 5개로 한정 → 위젯에서 항상 보이게.

### 슬라이드 5-3b. 대시보드 간트 — 색상 / 마일스톤 ◆ 의미
**막대 (프로젝트 상태)**
- 파랑 = 진행중 / **노랑 = 마감임박** / 빨강 = 이슈발생 / 초록 = 완료
- 좌측(진한 색) = 경과 기간 / 우측(60% 투명) = 예정 기간
**마일스톤 ◆ 다이아몬드 (긴급도 우선)**
- **빨간 ◆** = D-7 이내 임박 (긴급)
- **노란 ◆** = D-30 이내 임박 (주의)
- 핑크 ◆ = 단계(Phase) / 보라 ◆ = 셋업 작업 / 파랑 ◆ = 납기일
- 지난 마일스톤 = 40% 흐림 / hover = 이름·날짜·D-N 툴팁
- **노트:** 노란색은 두 가지 의미 — 막대=마감임박 상태, 다이아몬드=D-30 이내. 차트 위 범례 줄에 상시 표시.

### 슬라이드 5-4. 분석 / 회의록 모달 분리
- 헤더 버튼 **고급 분석 / 회의록 분리** (이전엔 한 모달에 섞여 있었음)
- 회의록 모달 = **프로젝트별 그룹** (인디고 헤더 + 내부 amber 타임라인 브랜치)
- **기본 모두 접힘**, 헤더 클릭 시 펼침. 우측 ↗ = 해당 프로젝트 회의록 탭으로
- 검색 / 프로젝트 / 작성자 필터 + "전체 펼치기/접기"
- **노트:** 새로 추가된 회의록 50건이 종합 Excel에도 별도 시트로 들어간다.

## 6장. 프로젝트 관리 — 핵심 워크플로우

### 슬라이드 6-1. 프로젝트 생성 (3단계)
1. 좌측 메뉴 → 프로젝트 관리 → **새 프로젝트**
2. 도메인(반도체/디스플레이/2차전지) 선택 — **셋업 일정 + 검수표 자동 세팅**
3. 메인 PM(인력 풀에서) 선택 → 저장
- 시작일/납기일은 **"미정" 체크박스**로 비워둘 수 있음
- **노트:** 도메인은 사후 변경이 ADMIN 전용. 처음부터 정확히.

### 슬라이드 6-2. 프로젝트 상세 — 7개 탭
- **일정** (단계별 / 셋업 두 서브탭) — 단계 편집 / 셋업 편집 모달
- **참고 자료** — 명세서/도면/기타 (Drive 폴더 자동 분류)
- **회의록** — 빠른/상세 모드 + 다중 파일
- **검수표** — Buy-off
- **추가 대응** — 워런티 단계 변경 추적
- **이슈 / 고객요청 / AS / 자재 / 버전**
- **이력** — 모든 활동 시간순 (클릭 시 해당 탭으로 점프)
- **노트:** 탭 전환은 좌측 칼럼. 모달은 **z-50 / 회의록·분석 모달은 z-150**.

### 슬라이드 6-3. 단계(Phase) 8단계
- 영업/수주 → 설계 → 구매/자재 → 제조/조립 → 출하 → **현장 셋업** → **워런티** → **완료**
- 검수 사인 시 자동 워런티 진입
- 단계 칩 클릭 = 임의 이동
- **노트:** 마지막 단계 = 완료 처리 단계. 이름 바꿔도 마지막이 완료로 인식됨.

### 슬라이드 6-4. 프로젝트 정보 수정
- 리스트의 **프로젝트명 / 고객사 셀** 클릭 = 수정 모달 (ADMIN/PM)
- 이름·고객사·사이트·일정·담당자 모두 수정
- **장비 코드 관리** (코드/모델/일련번호) — 파란 배지로 4개까지 노출 + "+N"
- **2차전지 (사이클러/EOL)** — 전압/전류/사양 추가 (보라 배지)
- 산업군 수정은 **ADMIN 전용** (활동 로그 자동 기록)
- **노트:** 장비 코드는 Enter로 빠른 추가. 한 프로젝트에 여러 코드 가능.

### 슬라이드 6-5. 담당자 / 출장 일정 통합 모달
- 리스트 **담당자 셀 클릭** = 한 모달에서 모두 처리
- ① 메인 PM 변경 (사유 기록, 이력 자동)
- ② 추가 인력 멀티 배정 (체크박스)
- ③ 출장 일정 — 한 사람이 같은 프로젝트에 여러 번 등록 가능
- ④ 출장 수정 = 변경 전/후 + 사유 + 수정자/일시 자동 이력
- **노트:** 출장 = 인력 화면과 대시보드에 자동 반영 (현장 파견 / 출장 예정 / 복귀 N일전).

## 7장. 단계 / 일정 / 간트차트

### 슬라이드 7-1. 일정 탭 — 두 모달 패턴 통일 (개편)
- 일정 탭 안에 **[프로젝트 단계별] / [셋업 일정]** 두 서브탭
- 양쪽 모두 **헤더 박스 + [단계 편집] / [셋업 일정 편집] 버튼**
- 표는 읽기 전용 — 직접 수정으로 인한 사고 방지
- **노트:** 클릭하다 실수로 데이터 바꾸는 일이 없도록 의도적으로 모달로만 편집.

### 슬라이드 7-2. 단계 편집 모달 (PM 이상)
- 이름 변경 / 추가 / 삭제 / 위↑↓ 순서
- **단계마다 시작일/종료일 직접 지정**
- **균등 분배** — 프로젝트 시작~종료를 단계 수로 자동 분배
- **일정 비우기** — 모든 단계 일정 초기화 (간트는 균등 분배 폴백)
- **마일스톤(SOP) 토글** — 켜면 간트에 빨간 ◆ + SOP 라벨
- 모든 변경은 활동 이력에 `PHASE_DEFINE`(추가 N · 수정 M · 삭제 K)로 자동
- **노트:** 셋업 일정 모달도 동일 UX (`SETUP_DEFINE`).

### 슬라이드 7-3. 간트차트 보기
- 우측 상단 **리스트 / 간트차트 토글**
- **단계별 탭** — 각 프로젝트 한 행, 막대 8단계 색 분할
- **셋업 일정 탭** — 프로젝트별 그룹, 헤더에 담당자/단계/진척률/작업수
- **마일스톤** — 빨간 ◆ + 종료일 라벨 (D-7 이내 펄스)
- **출장 일정** — 인디고 막대 자동 표시
- **오늘 표시** — 빨간 점선 + "▼ 오늘" 알약
- **노트:** 출장이 간트에 함께 보이는 게 핵심. 인력 분산 한눈에.

### 슬라이드 7-4. 간트 조작
- **휠 = 줌 (0.5x ~ 4x)**, **Shift+휠 = 가로 이동**
- ZoomIn/Out/오늘 버튼 / 차트 진입 시 today-1개월 위치 자동
- 좌측 칸 = 가로 스크롤 무관 / 월/일 헤더 = sticky-top
- **노트:** 조작 단축키는 직접 시연 권장.

### 슬라이드 7-5. 프로젝트 다중 필터
- 간트 좌상단 **"프로젝트 필터"** = 체크박스 드롭다운
- 검색 + 전체 선택/해제 + 부분 선택 시 "전체 보기로 초기화" 버튼
- 프로젝트 관리 우측 상단에 **도메인 필터** (반도체/디스플레이/2차전지)
- 담당자 필터와 AND 조건 — 리스트/간트 양쪽 적용
- **노트:** 필터를 잘 쓰면 "내가 봐야 할 것"만 정확히 본다.

## 8장. 회의록 / 참고자료

### 슬라이드 8-1. 회의록 — 빠른 / 상세 모드
- 프로젝트 상세 → **회의록 탭** → "+ 새 회의록 작성" 버튼만 노출
- 클릭 시 폼 펼침, 등록 후 자동 접힘 (리스트가 즉시 보임)
- **빠른 모드** — 본문 + 한줄 요약 + 첨부
- **상세 모드** — 회의 일시 / 참석자 / 결정사항 / 액션 아이템 추가
- **다중 파일** — 드래그/클릭, 진행률 `2/3 · 45%`
- **노트:** 빠른 모드 = 일상 미팅 메모, 상세 모드 = 정식 회의록.

### 슬라이드 8-2. 회의록 = 타임라인 브랜치
- 좌측 amber 세로선 + 캘린더 스타일 날짜 노드(월/일/요일)
- 색상은 신선도(최근=진하게, 오래된 것=흐리게)
- 작성자/시간/상대시간/요약/본문/결정/액션/첨부
- 대시보드 회의록 모달도 **프로젝트별 그룹 + 동일 타임라인**
- **노트:** 회의록 첨부 = `[프로젝트]/회의록` 폴더로 자동.

### 슬라이드 8-3. 참고자료 (회의록과 분리)
- 명세서·도면·기타 자료 = **참고 자료 탭** (회의록 탭과 별도)
- 업로드 시 **카테고리 선택** → Drive `[루트]/[고객사]/[프로젝트]/[카테고리]` 자동 분류
- 카테고리 필터 칩으로 그룹별 보기
- 단일 파일 최대 18MB (GAS 한도) — 큰 파일은 Drive 직접 업로드 후 링크
- **노트:** 사전 작업 = 시스템 설정에서 Drive 루트 폴더 등록 + 연결 테스트.

### 슬라이드 9-1. 검수표 작성 → 사인

- **최종 보고서 PDF** = 검수표 탭에서 인쇄/저장 (고객사 송부용)
- **노트:** 사인 취소는 신중하게. 모든 행위가 기록된다.

## 10장. 이슈 / 고객요청 / AS / 추가 대응

### 슬라이드 10-1. 이슈 / 펀치 관리
- 좌측 메뉴 → 이슈/펀치 관리 → "현장 이슈 등록"
- 모바일 모드 = 사진 첨부 + 빠른 등록
- 중요도 High/Medium/Low 자동 색상
- **이슈 제목/담당자 수정 = ADMIN 단독** (변경 전/후 + 사유, 자동 이력)
- **노트:** 작성 후 24시간 이상 지난 항목 수정은 사유 필수.

### 슬라이드 10-2. AS 통합 관리 + 고객 요청
- 좌측 메뉴 **"AS 통합 관리"** = 전 프로젝트 AS 한 화면 (AS 부서용)
- 검색 + 프로젝트/유형/상태 3중 필터
- AS 카드 클릭 = **AS 탭이 활성화된 채로** 모달 (이전엔 일정 탭)
- 모달 닫으면 AS 페이지에 그대로 남음 (이전엔 강제 전환되던 것 수정)
- **고객 요청** = 프로젝트별 고객요청 탭. 고객사도 직접 등록 + 응답 댓글 누적
- **노트:** AS = 운영, 고객 요청 = 신규 요구. 구분 필수.

### 슬라이드 10-3. 추가 대응 작업 (워런티 후)
- 검수 완료 후 **기능 추가 / 개선 / 버그 / UI / 공정 튜닝** 추적
- 셋업 일정과 분리 = 워런티 변경사항 명확히
- 상태: 대기 → 검토중 → 진행중 → 완료/반려
- **노트:** 고객 요청 탭 = 단순 요청 기록, 추가 대응 탭 = 실제 개발/구현. 혼동 금지.

## 11장. 인력 / 리소스

### 슬라이드 11-1. 엔지니어 등록 + 활동 이력
- 좌측 메뉴 → 인력/리소스 → "엔지니어 추가"
- 이름 / 직급 / 부서 / 위치 / 수동 상태
- 활성 프로젝트 체크박스로 배정
- 리스트 우측 **시계 아이콘** = 활동 이력 모달 (출장/이슈/노트/버전/AS/요청/PM변경 통합 타임라인)
- **노트:** 매칭 기준은 이름. 본인 명의 항목만 자동 집계.

### 슬라이드 11-2. 자격 정보 (출입증/안전/비자)
- 리스트의 **"자격/만료" 셀** 클릭 = 자격 관리 모달 (3 탭)
- 한 사람이 여러 출입증 / 여러 안전교육 / 여러 비자 가능
- **안전교육 = 만료없음 옵션** (상시 교육 — "상시 (만료없음)" 배지)
- 만료 = 빨간 배지 / 30일 이내 임박 = 노란 배지 자동
- **노트:** 비자 "필요/만료" = 비자 이슈로 자동 카운트.

### 슬라이드 11-3. 한눈에 보기 — 3컬럼 + 알림
- 페이지 상단이 대시보드 모달과 같은 **풍부 레이아웃**으로 동기화
- **6장 통계** — 전체 / 가용 / 현장 파견 / 출장 예정 / 자격 / 비자
- **3컬럼 그리드**
  - 현장별 인력 배치 — 사이트별 그룹, 복귀 D-day
  - 출장 일정 — 예정(D-7 강조) + 복귀 임박(30일)
  - 가용 인력 풀 — 이름+직급+부서+"대기" 배지
- **인력 알림 (풀 폭)** — 자격/비자 이슈 카드
- 헤더 우측 **"8주 가용성"** 버튼 → 히트맵 모달 (평소엔 가볍게)
- 하단 = 엔지니어 상세 테이블 (CRUD)
- **노트:** "다음달 셋업 누구 보낼까"는 8주 히트맵 모달로. 일상 운영은 3컬럼만 봐도 충분.

## 12장. 버전 관리

### 슬라이드 12-1. 프로젝트별 버전 등록
- 프로젝트 리스트의 **버전 컬럼 셀** 클릭 = 모달
- 카테고리 (HW/SW/충방전기 FW/인터페이스 FW 등 — 추천 칩) → 버전 → 배포일 → **변경 노트 (필수)**
- 카테고리/버전 칸 Enter, 노트 칸 Ctrl+Enter로 빠른 등록
- 모달 안 닫힘 = 카테고리 유지하며 연속 등록
- **노트:** 변경 노트 비우면 "이력을 입력하세요" 에러. 추적성 강제.

### 슬라이드 12-2. 카테고리 통일 / 전사 통합 뷰
- 카테고리 인덱스 = 도메인 추천 순서로 **자동 정렬** (예: HW → SW → 충방전기 FW → 인터페이스 FW)
- 자유 입력 카테고리는 추천 다음에 한글 순
- 좌측 메뉴 **"버전 릴리즈 관리"** = 전 프로젝트 버전 변경 시간 역순 통합
- **노트:** "최신 버전이 뭐였지?" = 카테고리 필터로 5초 안에.

## 13장. ★ 주간 업무 보고 [신규]

### 슬라이드 13-1. 왜 이 기능을 쓰나
- 매주 월요일 보고서 작성 시간 = 평균 30분 이상
- EQ-PMS 한 주간 활동을 **자동 집계**해서 보고서 초안 생성 → **수기 작성 90% 단축**
- 워드/엑셀 따로 X — 시스템 안에서 작성→제출→승인까지
- 팀장은 부서원 보고 받기 + 팀 종합 보고서 자동 작성
- **노트:** 시연이 가장 효과적. "지금 자동 작성 누르면 5초에 완성" 직접 보여주기.

### 슬라이드 13-2. 활성화 / 권한 (ADMIN)
- 시스템 설정 → **주간 업무 보고 기능 ON** → 전체 설정 저장
- 사용자 관리:
  - **"주간 보고" 토글** = 사용자별 사용 권한
  - **"팀장" 토글** = 부서장 권한 (PM과 별도, 같은 부서원 보고 검토)
- 활성화/권한 없으면 사이드바 메뉴 자체 숨김
- **노트:** 팀장 = PM과 무관. 팀장만 부서원 검토/승인 가능.

### 슬라이드 13-3. 자동 집계 — 어떤 활동이 어디로
| 섹션 | 들어가는 활동 |
|---|---|
| **금주 실적** | 회의록, 처리된 이슈/AS, 완료된 셋업, 검수 사인, 버전 릴리즈, 출장 |
| **차주 계획** | 다음 주 시작 셋업, 임박 마일스톤, 미해결 고객요청 |
| **이슈/리스크** | High 이슈, 지연 프로젝트, 자격 만료 임박, 중대 AS |
- 매칭 기준 = 본인 이름. 다른 사람 명의는 미포함.
- **노트:** "자동 집계 = 본인이 등록한 것만 잡힌다" 강조. 다른 사람이 처리한 건 자기 보고에 안 들어감.

### 슬라이드 13-4. 작성 워크플로우 (1분)
1. 좌측 메뉴 → **주간 업무 보고** → 주차 선택
2. **"100% 자동 작성"** 클릭 → 3개 섹션 자동 채움
3. 필요한 부분만 다듬기
4. **임시 저장** 또는 **제출**
5. 팀장 검토 → 승인/반려 (반려 시 사유는 작성자 화면 상단 붉은 배너)
- 단축 버튼: **"자동 + Excel"**, **"자동 + PDF"** = 자동 작성 → 추출 한 번에
- **노트:** 시연으로 보여주기. 30초만에 끝나는 것 강조.

### 슬라이드 13-5. 팀장 — 검토 + 팀 종합 보고서
- "팀원 보기" 탭 → **부서원 카드 클릭 = 같은 화면 모달**에서 보고+활동 근거 확인
- 모달 안에서 직접 **승인/반려(사유)** — 탭 이동 없음
- "팀 종합 보고서" 카드 클릭 = 부서원 보고 자동 통합 모달
  - 사람별 그룹 `【 김팀장 】 ... 【 홍길동 】 ...`
  - 4개 통계 박스(부서원/제출/승인/미제출) + 팀장 종합 코멘트 + 3섹션
  - 임시저장/제출/Excel/PDF
- **노트:** 부서원 보고가 추가/수정될 때마다 모달 다시 열면 자동 재집계.

### 슬라이드 13-6. 샘플 + 사용 가이드 (페이지 내장)
- 페이지 우측 상단 **"샘플 + 사용 가이드"** 버튼 = 6개 섹션 가이드
- 30초 사용법 / 자동 작성 결과 샘플 / 자동 집계 데이터 / 권한별 접근 / 워크플로우 / 팀 종합 보고서 시각 샘플
- **노트:** 처음 쓰는 사람도 샘플만 보면 양식·기대치 파악 가능.

## 14장. 활동 이력 / 알림

### 슬라이드 14-1. 활동 이력 = 이력 탭에서 클릭하면 점프
- 프로젝트 → 이력 탭 = 모든 활동 시간순

### 슬라이드 14-2. 알림 센터 활용 팁
- 출근 직후 5초 = 종 아이콘 → "어제 무슨 일이"
- 필터 칩으로 본인 관심 항목만 (예: AS만)
- "모두 읽음" 일괄 처리 + 마지막 본 시각 사용자별 저장
- **노트:** 알림은 푸시는 아니지만, 미확인 뱃지가 시각적으로 강함.

## 15장. 보고서 / Excel / PDF

### 슬라이드 15-1. Excel 추출 — 보고용 스타일
- 모든 Excel = **대시보드형 스타일** 자동
- 진한 블루 헤더 + 흰색 굵은 글씨 + 헤더 행 고정
- 자동 필터 / 한글 가중치 컬럼 너비 / 얼룩말 행
- 상태별 자동 컬러 (완료=초록, 지연/NG=빨강, 대기=노랑)
- 진행률(%) 자동 컬러, ISO 날짜 → YYYY-MM-DD
- **노트:** 별도 후처리 없이 그대로 보고용으로 송부 가능.

### 슬라이드 15-2. 추출 종류
- **대시보드 → 종합 리포트** = **차트 + 14시트** (기본통계/도메인/프로젝트별/지연/이슈/요청/AS/담당자변경/엔지니어/활동이력/마일스톤/임박이벤트/8주 히트맵/**최근 회의록 50건**)
- **대시보드 → PDF** = KPI 5장 + 임박 마일스톤 + 인력 임박 + 도메인별 진척률
- **프로젝트 관리** = 리스트(간단) / 상세(프로젝트별 시트 분리)
- **인력/리소스** = 3시트 / **이슈·자재·사이트·AS·버전** = 단독 추출
- **주간 업무 보고** = 본인 1시트 / 팀 종합 = 헤더+부서원별 시트
- **노트:** 메뉴마다 우측 상단에 Excel 버튼 — 같은 위치 일관.

### 슬라이드 15-3. PDF
- 검수표 탭 → **최종 완료 보고서 (PDF)** = 고객사 인수인계용
- 대시보드 → **PDF 출력** = 임원 보고용 1장 양식
- 주간 업무 보고 → **자동 + PDF** = 팀장/임원 보고용
- **노트:** 모두 인쇄 다이얼로그 → "PDF로 저장" 활용. 별도 라이브러리 없이 브라우저 인쇄.

## 16장. 권한별 빠른 참조

### 슬라이드 16-1. 일반 사용자(ENGINEER)가 매주 하는 일
- 출근 — 종 아이콘으로 신규 활동 확인
- 본인 배정 프로젝트의 **셋업 일정 진행** + 검수표 입력
- 회의록·이슈 등록 / AS 처리 / 버전 등록
- 출장 일정 확인 (자동 반영)
- 금요일 — **주간 업무 보고 자동 작성 → 제출**
- **노트:** 하루 5분, 주간 1분이면 충분.

### 슬라이드 16-2. PM / 팀장이 매주 하는 일
- 대시보드 KPI 5장 점검 (특히 지연·위험 / 고객 요청 / 미해결 이슈)
- 프로젝트 진척률 점검, 단계 이동, 담당자/출장 조정
- 이슈 우선순위 / AS 부서 협력
- (팀장) 부서원 주간 보고 검토·승인
- (팀장) 월요일 — **팀 종합 보고서 자동 작성 → 제출**
- **노트:** 팀 종합 보고서도 5~10분이면 충분.

## 17장. 자주 묻는 질문 (FAQ)

### 슬라이드 17-1. FAQ 1/2
- **Q. 시작일/납기가 정해지지 않았는데?**
  → "미정" 체크박스로 비워두기. 간트차트도 안전 처리.
- **Q. 도메인을 잘못 골랐어요.**
  → ADMIN에게 요청 (활동 로그 자동 기록).
- **Q. Buy-off 했는데 검수 항목 수정해야 해요.**
  → ADMIN만 가능. 사인 취소도 ADMIN.
- **Q. 모바일에서 사진 빠르게 첨부?**
  → 모바일 모드에서 이슈 등록. 자동 모드 전환 768px 미만.
- **Q. 같은 사람이 한 프로젝트에 출장 여러 번?**
  → 가능. 사유와 함께 모두 기록.

### 슬라이드 17-2. FAQ 2/2
- **Q. 주간 보고를 다른 사람이 볼 수 있나요?**
  → 본인 + 팀장(같은 부서) + ADMIN. 그 외엔 안 보임.
- **Q. 팀장인데 옆 부서 보고도 보고 싶어요.**
  → 같은 dept만 가능. 다른 부서는 ADMIN 권한 필요.
- **Q. 자동 집계가 빠진 것 같아요.**
  → 본인 이름으로 등록된 활동만 잡힘. 다른 사람이 한 일은 안 들어감.
- **Q. 사이드바에 주간 보고 메뉴가 안 보여요.**
  → ① 시스템 설정에서 기능 ON 됐는지 ② 사용자 관리에서 본인 "주간 보고" 토글 ON 됐는지.
- **Q. 첨부 18MB 넘는 파일은?**
  → Drive에 직접 올리고 링크 공유.

## 18장. 마무리

### 슬라이드 18-1. 오늘 핵심 5가지
1. 기록은 EQ-PMS 한 곳에 — 엑셀/메일 분산 X
2. 간트는 휠로 줌, Shift+휠로 이동
3. 회의록 = **빠른/상세 모드 + 다중 파일**
4. 주간 보고 = **자동 작성** + 1분 다듬기
5. 모르겠으면 헤더의 **❓ 도움말** (이 가이드 그대로)
- **노트:** 다음 주 첫 주간 보고는 함께 시연 후 각자 작성하는 시간 권장.

### 슬라이드 18-2. 지원 / 문의
- 시스템 문의 — AX 혁신실 / 본 프로젝트 담당자
- 권한/계정 — ADMIN
- 버그·개선 요청 — 사내 채널 또는 직접 등록
- 피드백 환영 — 신규 기능은 사용 패턴 기반으로 추가
- **노트:** Q&A 시간으로 마무리. 실제 본인 데이터로 한 번씩 들어가 보도록.

## 부록 A. 화면 캡처 가이드 (PPT 작성 시 참고)

PPT 슬라이드별로 권장 캡처 위치:
- **5-1, 5-2, 5-3**: 대시보드 상단 KPI 5장 + 우측 위젯
- **6-2**: 프로젝트 상세 모달 탭 메뉴
- **7-3, 7-4**: 간트차트 (단계별/셋업 양쪽)
- **8-1, 8-2**: 회의록 입력 폼(빠른/상세) + 등록된 회의록 타임라인
- **9-1**: 검수표 + Buy-off 사인 영역
- **10-2**: AS 통합 관리 페이지
- **11-3**: 8주 가용성 히트맵
- **13-1, 13-4, 13-5**: 주간 업무 보고 페이지 + 자동 작성 결과 + 팀 종합 보고서 모달
- **14-1**: 이력 탭 (점프 가능 카드들)

각 캡처에 빨간 화살표/박스로 클릭 위치 강조 권장.

## 부록 B. 변경 내역 한 페이지 요약 (배포 직전 공지용)

이번 라운드 주요 추가/개선:

1. **★ 주간 업무 보고 신규** — 자동 집계 + 팀 종합 보고서 + 샘플 가이드
2. **사용자 관리** — 팀장 / 주간 보고 컬럼 토글
3. **시스템 설정** — 글로벌 sticky 저장 바 (모든 설정 한 번에)
4. **대시보드** — KPI 5장 재구성 (고객 요청 미처리 추가, AS는 우측 위젯으로)
5. **회의록** — 빠른/상세 모드 + 다중 파일 + 타임라인 브랜치 + 프로젝트별 그룹 모달
6. **이력 탭** — 항목 클릭 시 해당 탭으로 점프
7. **AS 통합 관리** — 항목 클릭 시 AS 탭 자동 활성, 페이지 그대로 유지
8. **간트** — 두 모달 패턴 통일, 마일스톤 ◆, 출장 자동 표시, 휠 줌 + Shift+휠 가로
9. **프로젝트 정보** — 장비 코드, 2차전지 스펙(전압/전류/사양), 도메인 사후 수정
10. **인력** — 8주 가용성 히트맵, 가용 인력 풀, 자격 만료없음 옵션
11. **참고자료 / 회의록 분리** — Drive 카테고리 폴더링
12. **알림 센터** — 헤더 종 아이콘으로 전사 신규 활동 모아보기

배포 공지 메일/슬랙에 그대로 붙여 사용 가능.
