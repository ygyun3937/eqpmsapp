# 자재/스페어파트 파이프라인 설계 명세서

> **버전:** v0.1 (초안)  
> **작성일:** 2026-05-14  
> **협의 대상:** 설계팀 · 구매팀 · 개발팀  
> **목적:** 자재 생애주기(설계→구매→QC→제조→고객납품) 추적 시스템 설계 확정

---

## 1. 배경 및 문제 정의

### 현재 문제점

```mermaid
flowchart LR
    A[설계] -- "이력 단절 ❌" --> B[구매]
    B -- "QC 미실시 ❌" --> C[제조]
    C -- "이슈 뒤늦게 발견 🔥" --> D[납품 지연]

    style A fill:#e0f2fe,stroke:#0284c7
    style B fill:#e0f2fe,stroke:#0284c7
    style C fill:#fee2e2,stroke:#dc2626
    style D fill:#fee2e2,stroke:#dc2626
```

| 문제 | 영향 |
|------|------|
| 설계→구매 단계 이력 단절 | 누가 언제 무엇을 했는지 추적 불가 |
| QC 게이트 없음 | 불량이 제조 단계까지 넘어가서 발견 → 비용·일정 손실 |
| 수기 문서 작업 과다 | QC 성적서, 납품확인서 매번 직접 작성 |
| 현재 위치 불투명 | 자재가 어느 단계인지 실시간 파악 불가 |

---

## 2. 목표

- **QC 강제 게이트**: QC 미완료 시 제조 단계 진입 물리적 차단
- **완전한 이력 추적**: 모든 단계 전환에 담당자·시각 자동 기록
- **최소 입력**: QR 스캔 + 탭 몇 번으로 단계 처리 완료
- **자동 문서 생성**: QC 성적서·납품확인서 AI 자동 생성

```mermaid
    end

    end

    subgraph S5["⑤ 고객납품"]
        E1["전자서명\n+ 납품확인서"]
    end

    S1 --> S2 --> S3 --> S4 --> S5
    S3 -- "불합격 → 반려" --> S2

    style S3 fill:#fef3c7,stroke:#f59e0b,stroke-width:2px
```

### 3-2. 자재 타입별 분기

이 시스템은 두 가지 자재 타입을 동일한 파이프라인으로 처리합니다.

```mermaid
flowchart TD
    REG[파트 등록] --> TYPE{자재 타입 선택}

    TYPE -->|설계외주형| EXT["자사 설계 → 외주 제작\n예: 커스텀 브라켓, 전용 부품"]
    TYPE -->|구매형| PUR["외부 공급처 구매\n예: 표준 부품, 상용 제품"]

    EXT --> PIPE["동일 파이프라인\n설계 → 구매 → QC → 제조 → 납품"]
    PUR --> PIPE

    PIPE --> NOTE["설계 단계:\n설계외주형 → 도면 첨부 필수\n구매형 → 사양서/카탈로그 첨부"]

    style EXT fill:#dbeafe,stroke:#2563eb
    style PUR fill:#d1fae5,stroke:#059669
```

## 4. QR 기반 워크플로우

### 4-1. QR 생성 → 부착 → 스캔 흐름

```mermaid
sequenceDiagram
    participant PM as 설계팀(PM/설계자)
    participant SYS as PMS 시스템
    participant QR as QR 라벨
    participant W as 작업자(폰)
    participant AI as AI 문서엔진

    PM->>SYS: 파트 등록 + 단계별 체크리스트 정의
    SYS->>QR: QR 코드 자동 생성
    QR->>PM: 인쇄 → 도면 + 실물에 부착

    Note over W,QR: 이후 모든 단계는 QR 스캔으로 시작

    W->>QR: 📱 스캔
    QR->>SYS: 파트 ID 전달
    SYS->>W: 현재 단계 + 할 일 표시

    alt QC 단계
        W->>SYS: 체크박스 탭 + 사진 촬영
        SYS->>AI: 체크리스트 결과 전달
        AI->>SYS: QC 성적서 자동 생성
        SYS->>W: ✅ 합격 / ❌ 반려 처리
    else 납품 단계
        W->>SYS: 고객 전자서명
        SYS->>AI: 납품 데이터 전달
        AI->>SYS: 납품확인서 자동 생성
    else 기타 단계
        W->>SYS: "완료" 버튼 1탭
        SYS->>W: ✅ 다음 단계 자동 전환
    end
```

### 4-2. QR 라벨 구성

설계 단계에서 파트를 등록하면 아래 형태의 라벨이 자동 생성됩니다.

```
┌─────────────────────────────────────────┐
│  PRJ-2026-001        PART-2026-001      │
│  파트명: XXX 모터 브라켓                │
│  도면번호: DRW-MAK-0042                 │
│  수량: 5EA  |  긴급도: High             │
│  프로젝트: ○○○ 장비 셋업               │
│                                         │
│  [██████████]    QC 체크리스트 미리보기 │
│  [██ QR ████]    □ 치수 검사            │
│  [██████████]    □ 표면 처리 확인       │
│                  □ 재질 확인            │
└─────────────────────────────────────────┘
```

- 도면에 삽입: 설계 도면 우측 상단 또는 표제란에 QR 삽입
- 실물에 부착: 내구성 스티커 라벨 출력 → 자재/박스에 부착

## 5. 단계별 상세

### 5-1. ① 설계 (담당: 설계팀)

```mermaid
flowchart LR
    A[파트 등록 시작] --> B[기본정보 입력\n파트명·도면번호·수량·프로젝트]
    B --> C[자재 타입 선택\n설계외주 / 구매]
    C --> D[단계별 체크리스트 정의\nQC 항목 커스터마이징]
    D --> E[QR 코드 자동 생성]
    E --> F[라벨 출력\n도면 삽입 + 실물 부착]
    F --> G[구매 단계로 이관]
```

**설계팀 입력 항목:**

| 항목 | 필수 | 설명 |
|------|------|------|
| 파트명 | ✅ | 품명 |
| 도면번호 | ✅ | DRW- 형식 권장 |
| 수량 | ✅ | EA 단위 |
| 자재 타입 | ✅ | 설계외주 / 구매 |
| 프로젝트 연결 | ✅ | 관련 프로젝트 선택 |
| QC 체크리스트 | ✅ | 검사 항목 정의 (최소 1개) |
| 도면 파일 | 권장 | PDF/DWG 첨부 |
| 긴급도 | 선택 | High / Medium / Low |

---

### 5-2. ② 구매 (담당: 구매팀)

```mermaid
flowchart LR
    A[QR 스캔 또는 시스템 접속] --> B[공급처 선택\n신규 또는 등록된 업체]
    B --> C[납기일 입력]
    C --> D[발주서 AI 자동 생성]
    D --> E{입고 확인}
    E -->|정상 입고| F[QC 단계로 이관]
    E -->|납기 초과/불량 입고| G[알림 발송\n재발주 또는 반려]
```

**구매팀 입력 항목:**

| 항목 | 필수 | 설명 |
|------|------|------|
| 공급처 | ✅ | 업체명 선택/입력 |
| 발주번호 | 자동 | 시스템 자동 채번 |
| 납기일 | ✅ | 목표 입고 날짜 |
| 단가/금액 | 선택 | 견적 금액 |
| 발주서 | 자동 | AI 자동 생성 |

### 5-3. ③ QC — 필수 게이트 🔒 (담당: 품질/구매팀)

> **이 단계를 완료하지 않으면 제조 단계로 진입할 수 없습니다.**

```mermaid
flowchart TD
    A[QR 스캔] --> B[QC 체크리스트 표시\n설계 시 정의된 항목]
    B --> C[항목별 합격/불합격 탭]
    C --> D[사진 촬영\n불량 부위 또는 완료 증빙]
    D --> E{모든 항목 합격?}

    E -->|✅ 전체 합격| F[QC 성적서 AI 자동 생성]
    F --> G[제조 단계로 이관\n자동 알림 발송]

    E -->|❌ 1개라도 불합격| H[불합격 항목 메모 입력]
    H --> I[구매 단계 반려\n담당자 알림]
    I --> J[재발주 또는 공급처 클레임]

    style E fill:#fef3c7,stroke:#f59e0b,stroke-width:2px
    style G fill:#d1fae5,stroke:#059669
    style I fill:#fee2e2,stroke:#dc2626
```

**QC 체크리스트 예시 (자재 타입별):**

| 설계외주형 예시 | 구매형 예시 |
|----------------|------------|
| □ 치수 검사 (도면 대비) | □ 외관 검사 |
| □ 표면 처리 확인 | □ 규격/사양 확인 |
| □ 재질 확인 | □ 수량 확인 |
| □ 가공 정밀도 | □ 포장 상태 |
| □ 도면 번호 일치 여부 | □ 납품서 확인 |

## 7. 역할별 권한 및 책임

```mermaid
flowchart TD
    subgraph 설계팀
        D1[파트 등록]
        D2[QR 생성 및 배포]
        D3[도면 첨부]
        D4[체크리스트 정의]
    end

    subgraph 구매팀
        P1[발주 처리]
        P2[입고 확인]
        P3[QC 수행]
        P4[공급처 클레임]
    end

    subgraph 제조팀
        M1[제조 착수/완료]
        M2[조립 기록]
    end

    subgraph PM
        PM1[전체 현황 모니터링]
        PM2[납품 처리]
        PM3[보고서 확인]
    end

    설계팀 -->|파트 이관| 구매팀
    구매팀 -->|QC 합격 후 이관| 제조팀
    제조팀 -->|완료 후 이관| PM
```

| 역할 | 가능한 작업 |
|------|------------|
| 설계팀 (PM/ENGINEER) | 파트 등록, QR 생성, 도면 첨부, 체크리스트 정의 |
| 구매팀 (PM) | 발주 처리, 입고 확인, QC 수행, 반려 처리 |
| 제조팀 (ENGINEER) | 제조 착수/완료 처리 |
| PM/ADMIN | 전체 현황 조회, 납품 처리, 문서 확인 |
| CUSTOMER | 납품 서명, 본인 프로젝트 현황 조회 |

## 9. 기존 시스템과의 연계

```mermaid
flowchart TD
    subgraph 기존["기존 프로젝트 파이프라인"]
        PP["PROJECT_PHASES\n영업→설계→구매/자재→제조/조립→출하→현장셋업→워런티→완료"]
    end

    subgraph 신규["신규 파트 파이프라인"]
        AP["PART_PIPELINE\n설계→구매→QC→제조→납품"]
    end

    기존 -->|"프로젝트의 '구매/자재' 단계에서\n파트 파이프라인 활성화"| 신규
    신규 -->|"납품 완료 시\n프로젝트 자재 현황 자동 업데이트"| 기존
```

- 기존 `PART_PHASES: ['청구', '발주', '입고', '교체완료']` → 현장 스페어파트 교체용으로 유지
- 신규 파트 파이프라인 → 신규 제작/구매 자재의 생애주기 관리

## 10. 입력량 비교 (Before / After)

| 단계 | 기존 방식 | 신규 방식 |
|------|-----------|-----------|
| 파트 등록 | 폼 전체 수기 입력 | 기본정보 + 체크리스트 선택 |
| 구매 처리 | 별도 문서 작성 | QR 스캔 → 공급처 선택 → 탭 |
| QC | 성적서 수기 작성 (30~60분) | QR → 체크박스 탭 → 사진 → **AI 자동 생성** (3분) |
| 제조 완료 | 보고서 수기 작성 | QR → 버튼 1탭 |
| 납품 | 확인서 수기 작성 | QR → 고객 서명 → **AI 자동 생성** |
