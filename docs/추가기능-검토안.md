# A. 도메인 계층화 (대분류 / 중분류)

## 1. 현황

### 기존 (Flat 구조)
```js
DOMAINS = ['반도체', '디스플레이', '2차전지 사이클러', '2차전지 EOL']
```
- 도메인이 단일 평면 리스트. 신규 분야 추가 시 항목 자체를 늘려야 함.
- `2차전지 사이클러`, `2차전지 EOL` 처럼 같은 모기업/2차전지 하위인데 별도 도메인으로 분산.

### 변경 요청
| 대분류 | 중분류 (확장 가능) |
|---|---|
| **2차전지** | EOL, ESS, 사이클러, … |
| **반도체** | 플라즈마, … |
| **디스플레이** | (중분류 추가 가능) |

## 2. 영향 범위

### 데이터 구조
```js
// 변경 전
project.domain = '2차전지 EOL';

// 변경 후
project.domain = '2차전지';        // 대분류
project.subDomain = 'EOL';         // 중분류 (선택, 자유 추가 가능)
```

### 영향 받는 파일/로직
| 파일 | 역할 | 변경 필요 |
|---|---|---|
| `src/constants/index.js` | `DOMAINS`, `BATTERY_DOMAINS`, `DOMAIN_VERSION_CATEGORIES`, `DOMAIN_TASKS`, `DOMAIN_CHECKLIST` | 키를 대분류로 재구성, 중분류는 시스템 설정에 동적 등록 |
| `src/components/modals/ProjectModal.js` | 도메인 선택 | 대분류 select + 중분류 select 2단계 (cascading) |
| `src/App.js` `handleAddProject` | 신규 프로젝트 생성 시 task/checklist 시드 | 대분류 기준으로 시드 + 중분류별 오버라이드 가능 |
| `DashboardView.js`, `ProjectListView.js` | 도메인 필터, 통계 | 대분류 필터 + 선택 시 중분류 하위 필터 |
| `WeeklyReportView`, 모든 도메인 표시 | 라벨 | "2차전지 · EOL" 같이 결합 표시 |
| `IssueListView.js` 등 도메인 필터 사용처 | 동일 | |
| **GAS 백엔드 시트** | `domain` 컬럼 | 마이그레이션: 기존 "2차전지 EOL" → domain="2차전지", subDomain="EOL"로 분리 |

### 마이그레이션 전략
| 기존 값 | → 신규 domain | → 신규 subDomain |
|---|---|---|
| `2차전지 사이클러` | `2차전지` | `사이클러` |
| `2차전지 EOL` | `2차전지` | `EOL` |
| `반도체` | `반도체` | `(빈값)` 또는 `일반` |
| `디스플레이` | `디스플레이` | `(빈값)` |

→ **앱 부팅 시 1회 자동 마이그레이션** 또는 **관리자가 수동 일괄 변환 도구** 제공.

## 3. 중분류 관리 UI

**시스템 설정 → "도메인 관리" 패널** 추가:
```
┌──────────────────────────────────────────────┐
│ 도메인 관리                                    │
├──────────────────────────────────────────────┤
│ 2차전지                              [+ 중분류] │
│   · EOL                                 [삭제] │
│   · ESS                                 [삭제] │
│   · 사이클러                            [삭제] │
│                                                │
│ 반도체                               [+ 중분류] │
│   · 플라즈마                            [삭제] │
│                                                │
│ 디스플레이                           [+ 중분류] │
│   (등록된 중분류 없음)                          │
│                                                │
│ [+ 새 대분류]                                  │
└──────────────────────────────────────────────┘
```

- 대분류는 관리자만 추가/삭제 (영향 큼)
- 중분류는 PM 이상 추가 가능
- 사용 중인 중분류는 삭제 불가 (안전장치)

## 4. 도메인별 자동 시드 (체크리스트/태스크) 정책

| 옵션 | 설명 |
|---|---|
| **옵션 A** | 대분류 단위로만 시드 (기존 4개 → 3개로 줄어듦) |
| **옵션 B** (권장) | 대분류 기본 시드 + 중분류별 오버라이드 옵션. 중분류 등록 시 "어떤 중분류 시드를 복사할지" 선택 |
| **옵션 C** | 중분류별 완전 독립 시드 (지금처럼) — 추가/삭제 번거로움 |

## 5. 예상 공수

- 영향 화면 전체 회귀 테스트: **0.5일**

# B. 추가 대응 — 파일 임포트 기능

## 1. 배경
기존 시스템 도입 전 정리해둔 추가 대응 작업 리스트(Excel 등)를 **일일이 타이핑 안 하고 파일 업로드로 일괄 등록**.

## 2. 지원할 파일 형식

| 형식 | 장점 | 단점 |
|---|---|---|
| **Excel (.xlsx)** | 가장 많이 사용. 컬럼 매핑 직관 | `xlsx` 라이브러리 필요 (이미 export에서 사용 중) |
| **CSV** | 가장 단순, 어디서나 열림 | 인코딩(UTF-8 BOM) 주의, 셀 내 줄바꿈 약함 |
| **TSV / 클립보드 붙여넣기** | Excel/시트에서 복사 → 바로 붙여넣기 | 별도 라이브러리 불필요 |
| **JSON** | 정확한 구조 | 사용자가 직접 만들기 어려움 |

**권장**: Excel(.xlsx) + 클립보드 붙여넣기(TSV) **두 가지** 동시 지원

## 3. 표준 템플릿 (다운로드 제공)

```
| name (필수) | type   | requester | startDate  | endDate    | status | note          |
|-------------|--------|-----------|------------|------------|--------|---------------|
| 캘리브레이션 추가 | 기능 추가 | LGES 김선임 | 2026-01-15 | 2026-02-20 | 진행중 | 1차 협의 완료     |
| UI 색상 변경     | UI 변경  | A전자 박과장 | 2026-02-01 |            | 대기   |               |
```

- 필수 컬럼: `name` (작업명)
- 선택 컬럼: 나머지 (빈칸 시 기본값 사용)
- `type`은 화이트리스트 (기능 추가/기능 개선/버그 수정/UI 변경/공정 튜닝/기타) — 매칭 안 되면 "기타"
- `status`도 화이트리스트 (대기/검토중/진행중/완료/반려) — 매칭 안 되면 "대기"

## 4. 임포트 플로우

```
[추가 대응 탭]
    │
    ▼
[버튼: "파일로 일괄 등록"]
    │
    ▼
┌─────────────────────────────────────┐
│ 1. 템플릿 다운로드 (또는 직접 작성)    │
│ 2. 파일 선택 (.xlsx / .csv) 또는      │
│    Ctrl+V로 시트 데이터 붙여넣기      │
│ 3. 미리보기 + 검증                    │
│    - 필수 컬럼 누락 빨간색            │
│    - 잘못된 값 노란색 + 자동 보정      │
│    - 정상 행 녹색 체크                │
│ 4. "N건 등록" 클릭                    │
└─────────────────────────────────────┘
    │
    ▼
[항목별 createdBy = "import:사용자명"]
[원본 행 번호 + 임포트 시각 metadata 저장 → 롤백 가능]
```

## 5. 위험 / 안전장치
- **중복 방지**: 동일 프로젝트 내 같은 name + 같은 createdAt(±1분) 행 발견 시 건너뜀 옵션
- **롤백**: 임포트 직후 "방금 등록한 N건 취소" 버튼 (30초 토스트)
- **부분 실패 허용**: 100건 중 5건 오류면 95건만 등록, 오류 5건은 다운로드 가능
- **권한**: PM/ADMIN만 임포트 가능

## 6. 다른 탭으로 확장 (선택)
같은 패턴 재사용 가능:
- 노트 임포트 (회의록 일괄)
- AS 이력 임포트 (과거 AS 데이터)
- 이슈 임포트
- 자재/부품 임포트

→ 1차에는 추가 대응만, 검증되면 다른 탭으로 패턴 복사 (각 0.5일).

## 7. 예상 공수
- Excel 파싱 + 컬럼 매핑 UI: **0.5일**
- 미리보기/검증 UI + 임포트 핸들러: **0.5일**
- 템플릿 다운로드 + 가이드 문서: **0.5일**
- 합계: **1.5일**

## 8. 의사결정 항목

- [ ] **B2**. 파일 형식 (.xlsx만 / .xlsx + 붙여넣기 / 셋 다)
- [ ] **B3**. 권한 (PM 이상 / 누구나)
- [ ] **B4**. 중복 처리 (건너뜀 / 사용자 선택 / 무조건 추가)

# C. 메일 송부 — 출장신청서 / 출장보고서 / AS 보고서

## 1. 배경
시스템에서 내용 작성 → "상신" 버튼 → **수신인/참조자에게 정해진 양식대로 메일 발송**.

## 2. 기술적 검토

### 2.1 메일 발송 수단
| 옵션 | 동작 | 장단점 |
|---|---|---|
| **GAS `MailApp.sendEmail`** (권장) | GAS가 등록자(배포자) Google 계정으로 발송 | ✅ 별도 인프라 불필요. ❌ 일일 한도 100~1500통 (Workspace 플랜) |
| **GAS `GmailApp.sendEmail`** | 같음 + 첨부/HTML 풍부 | ✅ 한도 동일. HTML 양식 자유로움 |
| 외부 SMTP (SendGrid 등) | 별도 서비스 가입 | ❌ 추가 비용, ❌ 인증/도메인 셋업 필요. ✅ 발송량 무제한 |

→ **GmailApp 권장** (Workspace 계정이면 일일 1500통, 전사 충분).

### 2.2 발신자 계정
- GAS 배포자(=시스템 관리자) 계정으로 발송됨.
- 사용자 본인 이메일로 발송하려면 **OAuth + 권한 위임** 필요 → 복잡도 ↑
- **타협안**: 발신자는 시스템 계정(`mak-pms-noreply@회사도메인`), 본문에 작성자 이름 명시. 답장은 작성자 메일로 reply-to 설정.

### 2.3 HTML 메일 양식 (예시: 출장신청서)
```
┌──────────────────────────────────────────────┐
│ MAK-PMS 출장 신청서                            │
│ ──────────────────────────────────────────── │
│ 신청자       │ 김철수 (제조기술팀)              │
│ 프로젝트      │ A전자 텍사스 Fab 신규 셋업       │
│ 출장지        │ US-Texas Fab2                  │
│ 기간          │ 2026-06-01 ~ 2026-06-15        │
│ 목적          │ Phase 4 셋업 작업 및 시운전       │
│ 비용 예상      │ 4,500,000원                    │
│ 비고          │ 현지 호텔 직접 예약 예정          │
│                                                │
│ [시스템에서 확인] (링크)                         │
│ ──────────────────────────────────────────── │
│ MAK-PMS 자동 발송 메일                          │
└──────────────────────────────────────────────┘
```

## 3. UI 설계 — "메일 상신" 패널

각 모달(출장 등록 / AS 완료 / AS 통합 페이지)에 **"메일로 상신"** 버튼 추가:

```
┌─ 출장 신청 상신 ─────────────────────────────┐
│ 수신인 (To)                                    │
│  [팀장@회사 ×] [+ 추가]                       │
│                                                │
│ 참조 (CC)                                      │
│  [본인@회사 ×] [+ 추가]                       │
│                                                │
│ 양식 미리보기                                   │
│ ┌──────────────────────────────────┐          │
│ │ (위 HTML 양식 미리보기 렌더)        │          │
│ └──────────────────────────────────┘          │
│                                                │
│ 추가 코멘트 (선택)                              │
│ [_____________________________________]       │
│                                                │
│ [취소]              [상신 → 메일 발송]         │
└────────────────────────────────────────────┘
```

### 수신인 입력 방식
| 옵션 | 설명 |
|---|---|
| **A**. 자유 입력 | 이메일 직접 타이핑 |
| **B**. 사용자 목록에서 선택 | 시스템 등록 사용자 드롭다운 |
| **C** (권장) | A + B 둘 다. 자주 쓰는 수신인 기본값 (팀장 자동, 본인 CC) |

### 디폴트 수신인 (자동 설정)
- 출장신청서 → 수신: 작성자 부서 팀장. 참조: 작성자 본인
- AS 보고서 → 수신: 프로젝트 PM + 고객 담당자. 참조: 작성자
- 출장보고서 → 수신: 부서 팀장. 참조: PM + 본인

## 4. 메일 송신 기록 (감사)
시트에 `MAIL_LOG` 추가:
```
| 발송시각 | 종류            | 작성자 | 수신     | 참조  | 프로젝트 ID  | 메일 제목       | 상태 |
|----------|----------------|--------|----------|-------|--------------|----------------|------|
| 2026-05-13 14:30 | 출장신청서 | 김철수 | 박팀장@a | 본인  | PRJ-2026-001 | [상신] 텍사스... | 발송 |
```
- 사용자가 같은 건을 두 번 누르지 못하도록 cooldown
- 발송 후 시스템 안에서 "이 보고서는 X월 X일 발송됨" 표시

## 5. 위험 / 제약
- **Workspace 외부 발신 차단 정책**: 회사 도메인에서 외부 메일 차단해두면 고객사로 못 보냄 → IT 확인 필수
- **개인정보**: 메일 본문에 들어가는 정보 (이름, 사이트, 비용 등) — 외부 메일은 평문 전송. 민감 항목은 시스템 링크로만 (메일에는 제목만)
- **첨부**: AS 보고서 PDF 첨부 가능 (GmailApp.sendEmail의 attachments 옵션). Drive 링크 첨부도 가능.

## 6. 예상 공수
- 양식 3개 (출장신청/출장보고/AS보고) HTML 템플릿 + 미리보기: **1일**
- 수신인/참조 입력 UI + 디폴트 자동 설정: **0.5일**
- GAS `sendEmail` 함수 + 로그 시트: **0.5일**
- 보낸 메일 추적 UI: **0.5일**
- 합계: **2.5일**

- [ ] **C4**. 첨부 (PDF 생성하여 첨부 / Drive 링크만)
- [ ] **C5**. 1차 적용 양식 (출장신청만 → 검증 → 나머지 확장 / 동시 3종)
