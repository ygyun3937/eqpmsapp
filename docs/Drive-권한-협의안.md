# Google Drive 폴더 접근 권한 협의안

> **목적**: 회의록 / 노트 / 참고 자료 / AS / 명세서 / 도면 등을 업로드하는 Drive 폴더의 **누가, 무엇을, 어디까지** 접근할 수 있게 할지 정책 합의.
>
> **상태**: 협의안 (의사결정 전).
>
> **작성일**: 2026-05-13

## 1. 현재 동작 요약

### 폴더 구조
```
[루트 폴더 (시스템 설정에 등록된 단일 ID)]
└── [고객사명] (예: A전자)
    └── [프로젝트명-PRJ-2026-001] (예: P3 셋업-PRJ-2026-001)
        ├── 명세서
        ├── 도면
        ├── 회의록
        ├── 노트
        ├── AS
        └── 기타
```
- 자동 생성: 업로드 시 고객사/프로젝트/카테고리 폴더가 없으면 생성.
- 파일명: 사용자 업로드 원본명 그대로.

### 접근 권한 (현재)
- **GAS 백엔드는 등록자(MAK-PMS 관리자) 계정 권한으로 파일을 생성** → 파일 소유자 = GAS 배포자.
- 명시적 권한 부여 코드는 주석 처리되어 있음 (`docs/gas-backend.gs:265`).
- **즉**, 루트 폴더의 공유 설정을 그대로 상속받음.
  - 루트 폴더가 "회사 도메인 전체 보기"면 → 회사 누구나 봄
  - 루트 폴더가 "특정 멤버만"이면 → 그 멤버만 봄
- 클라이언트(MAK-PMS)에는 `viewUrl` (드라이브 미리보기 링크)와 `downloadUrl` 반환. 사용자가 클릭하면 Google Drive 인증 거쳐 접근.

### 보안 관련 코드 위치
| 위치 | 내용 |
|---|---|
| `docs/gas-backend.gs:213` | `VERIFY_DRIVE_FOLDER` — 폴더 ID 유효성 검증 |
| `docs/gas-backend.gs:265` | `file.setSharing(...)` — 명시적 권한 부여 (현재 주석) |
| `src/App.js` `handleUploadAttachment` | 카테고리 결정 (`명세서`/`도면`/`회의록`/`노트`/`AS`/`기타`) |

## 2. 협의가 필요한 의사결정 항목

### A. 접근 대상 범위

**누가 Drive 파일에 접근할 수 있어야 하는가?**

| 옵션 | 설명 | 장점 | 단점 |
|---|---|---|---|
| **A1. MAK-PMS 사용자 전원** | MAK-PMS 로그인 계정을 가진 모든 사용자 (회사 내부) | 단순, 협업 원활 | 권한 분리 없음. 고객 정보 무차별 노출 가능 |
| **A2. 프로젝트별 권한** (권장) | 프로젝트에 배정된 엔지니어/PM/팀장만 해당 프로젝트 폴더 접근 | 최소 권한 원칙 부합 | 폴더별 권한 부여 자동화 필요. 누락 시 사용자 혼란 |
| **A3. 부서별 권한** | 같은 부서원 + 관리자만 | 운영 단순 | 부서 간 협업 막힘. 인사 이동 시 권한 갱신 필요 |
| **A4. 고객사별 권한** | 같은 고객사 담당 인력 + 관리자만 | 고객사 보안 격리 | 동일 고객사 内 PM/엔지니어 권한 차이 못 둠 |

**결정 필요**: A1 / A2 / A3 / A4 중 어느 것?

### B. 고객(외부) 공유 정책

**고객사 담당자(외부 이메일)에게 일부 파일을 공유할 일이 있는가?**

| 옵션 | 설명 |
|---|---|
| **B1. 일절 외부 공유 금지** | 외부 공유 시도 자체를 차단 (Drive 도메인 정책 + GAS 코드 강제) |
| **B2. 명시적 공유만 허용** (권장) | 기본은 비공개, 사용자가 "외부 공유" 버튼 클릭 → 명시적으로 이메일 추가 |
| **B3. 링크 보유자 보기 허용** | 링크만 있으면 누구나 보기 가능 (현재 주석 처리된 코드의 동작) |

**결정 필요**: 외부 공유 방식?

### C. 카테고리별 권한 차등

회의록 / 노트 / 참고 자료는 내부용. **명세서 / 도면 / AS 보고서는 고객 공유 빈도가 더 높음**.

**카테고리별로 권한을 다르게 둘 것인가?**

| 옵션 | 설명 |
|---|---|
| **C1. 동일 권한** | 카테고리 무관 동일 (단순) |
| **C2. 내부/외부 분리** (권장) | "내부용"(회의록·노트·기타)과 "외부 공유 가능"(명세서·도면·AS 보고서) 분리. 외부 공유는 후자만. |

**결정 필요**: C1 / C2?

### D. 권한 갱신 책임

**누가 프로젝트 멤버 권한을 추가/제거하는가?**

| 옵션 | 설명 |
|---|---|
| **D1. ADMIN만** | 관리자가 수동으로 폴더 권한 관리 |
| **D2. PM이 자기 프로젝트만** (권장) | 프로젝트 담당 PM이 멤버 추가/제거 (MAK-PMS UI 안에서) |
| **D3. 자동 동기화** | MAK-PMS의 프로젝트 배정 변경 시 Drive 권한 자동 갱신 (GAS 스크립트로) |

**결정 필요**: 누가, 어떻게?

### E. 권한 만료 / 회수

**사용자가 퇴사하거나 프로젝트에서 빠지면 권한은 어떻게 회수?**

| 옵션 | 설명 |
|---|---|
| **E1. 수동 회수** | 관리자가 인지하면 회수 (현재 기본) |
| **E2. 자동 회수** (권장) | MAK-PMS에서 "비활성화" 처리 시 Drive 권한도 자동 회수 (GAS 스크립트) |
| **E3. 기간 만료** | 매 분기 또는 매년 전체 권한 재확인 (Drive Audit) |

**결정 필요**: 회수 방식?

### F. 감사 / 로깅

**누가 언제 무엇을 다운로드했는지 추적해야 하는가?**

| 옵션 | 설명 |
|---|---|
| **F1. Google Workspace 기본 감사 로그만** | Workspace Admin Console에서 조회 (Business Standard 이상) |
| **F2. MAK-PMS 자체 로그** | 다운로드/조회 시 자체 시트에도 기록 (별도 구현 필요) |
| **F3. 둘 다** | Workspace 로그 + 자체 로그 |

**결정 필요**: 로깅 범위?

### G. 외부 PC / 모바일 접근

**회사 외부 네트워크나 개인 PC/폰에서 접근 가능해야 하는가?**

| 옵션 | 설명 |
|---|---|
| **G1. 회사망/회사기기만** | Workspace Context-Aware Access로 IP/디바이스 제한 |
| **G2. 인증되면 어디서나** (권장 — 현장 작업 고려) | Google 계정 인증 통과 시 어디서나 |
| **G3. 회사망만 + VPN** | 현장은 VPN 필수 |

**결정 필요**: 접근 환경 제한?

## 3. 권장 정책 (참고 — 협의에 가져갈 시안)

> 현장 운영 효율과 최소권한 균형:

- **A2** (프로젝트별 권한) — 배정된 인력만 폴더 접근
- **B2** (명시적 외부 공유) — 기본 내부, 필요 시 사용자가 이메일 단위 추가
- **C2** (카테고리 분리) — 외부 공유는 명세서/도면/AS 보고서만
- **D2** (PM 책임) — PM이 자기 프로젝트 멤버 관리
- **E2** (자동 회수) — 비활성화/프로젝트 이탈 시 권한 자동 제거
- **F1** (Workspace 감사) — 자체 로그는 다음 단계
- **G2** (어디서나 + Google 인증)

## 4. 기술적 검토 사항 (협의 후 구현 시)

### 4.1 자동 권한 부여 / 회수 구현 난이도

- **DriveApp API**: 폴더 단위로 `addEditor(email)`, `removeEditor(email)`, `addViewer(email)` 호출 가능. 단순.
- **트리거 위치**:
  - 프로젝트 배정 변경 시 (`assignedEngineerIds` 변경) → 권한 갱신
  - 사용자 비활성화 시 (`active: false`) → 모든 폴더에서 제거
  - 새 폴더 생성 시 → 현재 멤버 자동 등록
- **예상 공수**: 약 1일 (GAS 함수 3~4개 추가 + 클라이언트 트리거 연결)

### 4.2 Workspace 도메인 제약

Drive 폴더의 공유 정책은 Workspace 관리자가 도메인 단위로 막을 수 있음. 우선 확인:
- 회사 도메인에 **외부 공유 차단** 정책이 걸려 있는가?
- 걸려 있다면 옵션 B2는 사실상 불가능 (예외 신청 필요).

### 4.3 고객(외부 도메인) 계정 처리

- 고객이 Google 계정이 없으면? → Drive 공유 안 됨 → 다른 채널(이메일 첨부, S3 사전 서명 URL 등) 필요.
- 고객이 다른 Workspace 계정이면? → 도메인 간 공유 허용 정책 필요.

### 4.4 사진 첨부 용량

- AS 사진 / 도면 스캔 누적 → 프로젝트당 GB 단위 가능.
- Workspace 플랜(Business Starter 30GB/유저 → Standard 2TB)에 따라 정책 다름.
- 1년 후 용량 분석 후 보관/이관 정책 별도 협의.

## 5. 협의 시 결정 항목 체크리스트

회의에서 결정해야 할 사항을 한 페이지로:

- [ ] **A**. 접근 범위 (A1/A2/A3/A4)
- [ ] **B**. 외부 공유 정책 (B1/B2/B3)
- [ ] **C**. 카테고리별 차등 (C1/C2)
- [ ] **D**. 권한 부여 책임자 (D1/D2/D3)
- [ ] **E**. 권한 회수 방식 (E1/E2/E3)
- [ ] **F**. 감사 로깅 (F1/F2/F3)
- [ ] **G**. 접근 환경 제한 (G1/G2/G3)
- [ ] **H**. Workspace 도메인 정책 확인 (외부 공유 막혀있는지)
- [ ] **I**. 고객 계정 처리 방침 (Google 없는 고객 시)
- [ ] **J**. 누가 GAS 폴더 ID 변경 권한? (관리자 1인 / 다수)
